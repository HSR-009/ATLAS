1. Introduction

1.1 Background

Organizing large-scale events such as college fests, workshops, donation drives, and conferences is a complex undertaking. Event organizers must simultaneously manage multiple departments, coordinate volunteers, assign and track tasks, and monitor overall progress — all while ensuring nothing falls through the cracks.

1.2 Problem Statement

Current event management approaches suffer from several critical shortcomings:

  • Fragmented Communication: Event data is scattered across multiple platforms (WhatsApp groups, Google Sheets, paper logs), making it difficult to maintain a single source of truth.
  • Lack of Real-Time Visibility: Organizers have no centralized dashboard to view overall event progress, pending tasks, or department-level status at a glance.
  • Manual Task Tracking: Task assignments are communicated informally, leading to ambiguity about responsibilities and deadlines.
  • No Audit Trail: There is no systematic log of actions performed — who assigned what, when was it completed, and by whom.

1.3 Proposed Solution

ATLAS is a native Android application designed to serve as a comprehensive, all-in-one event management platform. It provides organizers and volunteers with a centralized system to create events, organize departments, onboard team members, assign and track tasks, and maintain a complete activity log.

2. Objectives

  1. Centralized Event Management: Provide a single platform to create, view, and manage multiple events.
  2. Hierarchical Organization: Enable organizers to structure events into departments with custom identities.
  3. Task Assignment & Tracking: Allow creation of tasks with assignees, due dates, and priority levels.
  4. Volunteer Management: Support onboarding volunteers into specific departments with role designations.
  5. Activity Logging: Automatically record all significant actions with timestamps.
  6. Persistent Data Storage: Store all application data locally using SQLite via Room.

8.1 Performance Optimizations

  • Lazy Loading: Lists are rendered efficiently using Compose LazyColumns.
  • Thread Management: Room operations reside strictly on IO Dispatchers.
  • State Hoisting: Clean state management prevents unnecessary recompositions.
